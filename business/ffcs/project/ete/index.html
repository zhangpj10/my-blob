# 客服感知端到端监控系统

# 项目描述

  1. 开发工具：idea+plsql+sourcetree

  2. 技术栈：springmvc+java+gradle+mybatis+angularjs+oracle+elasticsearch+echars+twaver+vuejs

  3. 项目描述：该项目围绕用户感知，梳理重点业务流程，实现智慧型运营监控，主要负责模块包含端到端总体视图、预警看板、IT服务树、评估报告，助力电信在用户感知、运营效能等方面的提升。

  4. 主要负责

（1）总体视图：通过地图展示省份各地市及各县某个时间段的指标达标率，地图可配置；通过柱状图展示达标率TOP5；展示各个指标的达标率和阀值，并可拖拽指标项进行排序。

（2）预警看板：通过地图展示省份各地市及各县某个时间段的告警数，地图可配置；通过折线图和柱状图展示某区域的告警情况；对指标进行分类及展示各指标的告警情况。

（3）IT服务树：通过twaver将各个系统的数据响应情况串联起来，展示在页面上。

（4）评估报告：展示对应工单解决前和解决后的评估情况以及问题解决前后趋势的情况。

# 疑难杂症：

  1. echarts官网提供的地图不全，自己通过摸索地图json的数据格式，利用工具重新生成json文件

  2. echarts没有提供ui所要求的圆环进度条图（即图中的整个地市的指标达标率），后面通过采用h5的canvas实现。

  3. 在项目赶的情况下，需要去了解elasticsearch，进行各指标的取数和计算。
