    1. vue的双向绑定原理

    # 原理：

    主要是通过数据劫持结合发布者订阅者的模式来实现的。

    # 个人理解

    双向绑定，无非就是数据层与视图层之间的数据同步，视图层会便随数据的改变而实时更新。
    发布者-订阅者模式：getter函数里面执行的任务就是watcher订阅者，而setter函数执行的任务就是发布者。但是首先你要先了解数据属性和访问器属性，下面浅谈一下：

    # 数据属性

    • Configurable: 表示能否通过delete将属性删除，能否把属性修改为访问器属性，默认为false。当把属性Configurable设置为false后，该属性不能通过delete删除，并且也无法再将该属性重置回true。
    • Enumerable: 表示属性可否被枚举（即是否可以通过for in 循环返回），默认为false
    • Writable: 表示属性是否可写，默认false。
    • Value: 该属性的数据值，默认为undefined。

    # 访问器属性

    • Configurable: 表示能否通过delete将属性删除，能否把属性修改为访问器属性，默认为false。当把属性Configurable设置为false后，该属性不能通过delete删除，并且也无法再将该属性重置回true。
    • Enumerable: 表示属性可否被枚举（即是否可以通过for in 循环返回），默认为false。
    • get: 读取属性时调用的函数，默认为undefined。
    • set: 写入属性时调用的函数，默认为undefined。

    在修改属性的特性或者定义访问器属性的时候，需要借助Object.defineProperty()，这个方法接收三个参数，即属性所在对象，属性名字，描述符对象。

    # 示例代码

    let obj = {};
let oInput = document.getElementById('inp');
let oBox = document.getElementById('box');

Object.defineProperty(obj, 'name', {
    configurable: true,
    emumerable: true,
    get() {
        console.log('23');
        return val;
    },
    set(newVal) {
        oInput.value = newVal;
        oBox.innerHTML = newVal;
    }
});

oInput.addEventListener('input), (e) => {
    obj.name = e.target.value;
});

obj.name = '小明';

    所以实现数据双向绑定的核心就是利用为每个属性都创建订阅者的实例对象作为观察；getter函数会返回一个value值，当setter函数写入或者修改后，会调用update()方法更新视图的数据值，Configurable和Enmuberable这两个默认为true,可以不用写，示例代码如下：

    function defineReactive(obj, key, val) {
    const dep = new Dep(); // 这是一个构造函数，其原型为属性添加订阅者,最后面会有示例代码
    Object.defineProperty(obj, key, {
        get() {
            if (dep.target) {
                dep.addSub(dep.target); // 添加订阅者到Dep实例对象里
            }
            return val; // 返回箭头的value值
        },
        set(newVal) {
            if (newVal === val) return;
            val = newVal; // 写入新的value值
            dep.notify(); // 作为发布者发布通知，然后dep回迭代调用各自的update()方法来更新视图
        }
    }) 
}
function observe(obj, vm) {
    Object.keys(obj).forEach((key) => {
        defineReactive(vm, key, obj[key]);
    });
}

    # 条件：

    通过上面的理解，我们大概知道双向绑定需要的条件是什么了，如下：

    • 需要实现一个监听器Observer,对数据对象的所有属性进行监听，如果有属性变动了，就可以拿到最新值并通知订阅者。
    • 需要实现一个解析器Compile,主要是对Dom树的每个节点进行扫描和解析，如果数据有变就替换数据，以及绑定相应的更新函数。
    • 需要实现一个订阅器Watcher, 作为连接Observer和Compile的桥梁，能够订阅并接收每个属性变化的通知，执行指令绑定的相应回调函数，从而更新视图。

    # 双向绑定示例代码

    // 订阅和发布（依赖收集类）
class Dep {
    constructor() {
        this.subs = [];
    }
    addSub(sub) {
        if (sub && !this.subs.includes(sub)) {
            this.subs.push(sub);
        }
    }
    notify() {
        this.subs.forEach(sub => sub());
    }
}

let Person = {
    name: '张三',
    age: '22'
};
let target = null;
let info = '';

Object.keys(Person).forEach((key) => {
    let value = Person[key];
    let dep = new Dep();
    Object.defineProperty(Person, key, { // 数据劫持
    // 通过getter进行依赖收集，而每个setter方法就是一个观察者，在数据变更的时候通知订阅者更新视图
      get() {
        dep.addSub(target);
        return value;
      },
      set(val) {
        value = val;
        dep.notify();
      }
    })
})

// 封装一个watcher函数. 帮你观察记录相关target事件，避免多次声明变量
 function Watcher(myFunc) {   
    target = myFunc;
    target();
    target = null;
 }

Watcher(() => {
    info = `${Person.name}今年${Person.age}岁了`;
})

console.log(info);
Person.name = 'zhangpj';
Person.age = '28';
console.log(info)
